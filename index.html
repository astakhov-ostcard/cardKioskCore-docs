    Структура json файлов¶

    Базовая логика картомата и его внешний вид описывается в формате json.

    Apis - отправка запросов, интерпретация ответов (REST api)

    Windows - визуальные элементы, скомпонованные в экраны, которые видит пользователь

    Auxilary - работа с подключаемыми устройствами (сканеры, лотки и т.д.)

    Styles - наборы параметров отображения визуальных элементов

    Timers - задержки перед выполнением команд, периодическое выполнение команд

    Работа с переменными¶

    Для хранения информации используется два типа переменных:

    1. Sessiond data - хранение данных в рамках одной сессии (одного пользователя)

    2. Pesistent data - хранение данных в течении всей работы картомата (до перезагрузки)

    Операции с переменными - соединение и разделение строк, удаление, перезапись и т.д.

    Локализация¶

    Подстановка текста, ссылок и графики в зависисмости от выбранного пользователем языка. Перед началом работы потребуется понимание структуры json файлов и работы с переменными
