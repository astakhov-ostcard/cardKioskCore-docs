## Чтение значения

Для подстановки значения переменной в параметры элементов и команд используется префикс `session:`:

```json
"session:varName"
```

### Модификаторы

Добавляются через `|` после имени переменной:

| Модификатор | Описание | Пример |
|---|---|---|
| `\|fallback:текст` | Текст, если переменная не задана | `"session:CardNumber\|fallback:Не указан"` |
| `\|mask:????-????` | Форматирование по маске (`?` — символ значения) | `"session:Pin\|mask:????"` |

Модификаторы можно комбинировать:

```json
"session:CardNumber|mask:****-????-????-????|fallback:Нет данных"
```

### Вложенные пути (dot-path)

Если в переменную записан JSON-объект (например, результат API), к его полям можно обращаться через точку:

```json
"session:clientData.name"
"session:accountInfo.balance|fallback:0"
```

## Команды

### `add:` — Записать (всегда)

```json
"add:varName"
"add:varName#value"
```

- Без `#value` — записывает значение `true`
- С `#value` — разрешает значение и записывает результат
- **Всегда перезаписывает** существующее значение

---

### `fill:` — Записать (только если пусто)

```json
"fill:varName"
"fill:varName#value"
```

Аналог `add:`, но записывает значение **только если переменная ещё не существует** в SessionData.

---

### `remove:` — Удалить переменную

```json
"remove:varName"
```

---

### `pushSession:` — Скопировать значение из другой переменной

```json
"pushSession:sourceKey>targetKey"
```

Копирует значение переменной `sourceKey` (ищет сначала в Session, затем в Persistent) в `session:targetKey`.

---

## Сводная таблица команд

| Команда | Перезаписывает | Описание |
|---|---|---|
| `add:var` | Да | Записать `true` |
| `add:var#val` | Да | Записать значение |
| `fill:var` | Нет | Записать `true`, если пусто |
| `fill:var#val` | Нет | Записать значение, если пусто |
| `remove:var` | — | Удалить переменную |
| `pushSession:src>dst` | Да | Скопировать из любого источника в Session |
