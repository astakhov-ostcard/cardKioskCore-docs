# PersistentData

## Что это и зачем нужно

**PersistentData** — долговременное хранилище, которое живёт на протяжении всего времени работы приложения до перезагрузки. Данные не привязаны к конкретному пользователю и не сбрасываются при завершении сессии.

Используется для данных, которые должны сохраняться между сессиями: выбранный язык интерфейса, настройки устройства, счётчики операций, профиль оператора.

> Для данных, которые относятся к конкретному пользователю и должны очищаться после его ухода, используйте [SessionData](SessionData).

---

## Чтение значения

Для подстановки значения переменной в параметры элементов и команд используется префикс `persistent:`:

```json
"persistent:varName"
```

### Модификаторы

Добавляются через `|` после имени переменной:

| Модификатор | Описание | Пример |
|---|---|---|
| `\|fallback:текст` | Текст, если переменная не задана | `"persistent:appLanguage\|fallback:ru"` |
| `\|mask:????-????` | Форматирование по маске (`?` — символ значения) | `"persistent:deviceCode\|mask:????"` |

Модификаторы можно комбинировать:

```json
"persistent:deviceCode|mask:??-??-??|fallback:Не задан"
```

### Вложенные пути (dot-path)

Если в переменную записан JSON-объект, к его полям можно обращаться через точку:

```json
"persistent:deviceConfig.branchId"
"persistent:deviceConfig.location|fallback:Неизвестно"
```

## Команды

### `persistentfill:` — Записать (только если пусто)

```json
"persistentfill:varName"
"persistentfill:varName#value"
```

- Без `#value` — записывает значение `true`
- С `#value` — разрешает значение и записывает результат
- Записывает **только если переменная ещё не существует** в PersistentData

> Команды `persistentadd:` нет — запись в Persistent намеренно возможна только при отсутствии значения. Для перезаписи используйте `persistentremove:` + `persistentfill:`.

---

### `persistentremove:` — Удалить переменную

```json
"persistentremove:varName"
```

---

### `pushPersistent:` — Скопировать значение из другой переменной

```json
"pushPersistent:sourceKey>targetKey"
```

Копирует значение переменной `sourceKey` (ищет сначала в Session, затем в Persistent) в `persistent:targetKey`.

---

## Сводная таблица команд

| Команда | Перезаписывает | Описание |
|---|---|---|
| `persistentfill:var` | Нет | Записать `true`, если пусто |
| `persistentfill:var#val` | Нет | Записать значение, если пусто |
| `persistentremove:var` | — | Удалить переменную |
| `pushPersistent:src>dst` | Да | Скопировать из любого источника в Persistent |

---

## Важные особенности

- Без `#value` всегда записывается `true` — удобно для флагов первого запуска: `"persistentfill:isInitialized"`.
- Преднамеренное отсутствие `persistentadd:` защищает от случайной перезаписи долговременных настроек. Для обновления всегда используйте связку `persistentremove:` + `persistentfill:`.
- `persistent:` поддерживается как источник во многих параметрах элементов: `text`, `source`, `value`, а также в команде `language:`.
- Команды можно комбинировать с конкатенацией `&&` и разделением `.Split()` — подробнее в разделе [Операции с переменными](VariablesFunctions).
