## Структура json файлов

Базовая логика картомата и его внешний вид описывается в формате json. 

[Apis](Apis) - отправка запросов, интерпретация ответов (REST api) 

[Windows](Windows) - визуальные элементы, скомпонованные в экраны, которые видит пользователь

[Auxilary](Auxilary) - работа с подключаемыми устройствами (сканеры, лотки и т.д.)

[Styles](Styles) - наборы параметров отображения визуальных элементов

[Timers](Timers) - задержки перед выполнением команд, периодическое выполнение команд

## Работа с переменными

Картомат в процессе работы собирает и использует различные данные: номер карты, выбранную услугу, язык интерфейса, результат последней операции и многое другое. Чтобы эти данные были доступны на разных экранах и в разных командах, они хранятся в двух специальных хранилищах.

1. [Sessiond data](SessionData) - хранение данных в рамках одной сессии (одного пользователя). Автоматически удаляются при переходе на стартовый (root) экран. Используется для данных, которые уникальны для каждого пользователя: номер карты, введённый PIN, результаты запросов.  
2. [Pesistent data](PersistentData) -  — долговременное хранилище, которое живёт на протяжении всего времени работы приложения до перезагрузки. Используется для данных, которые не зависят от конкретного пользователя: выбранный язык интерфейса, настройки устройства, счётчики операций.  

[Операции с переменными](VariablesFunctions) - соединение и разделение строк, удаление, перезапись и т.д.
