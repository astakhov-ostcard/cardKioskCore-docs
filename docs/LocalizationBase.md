# Локализация — Принцип работы

## Обзор

Локализация позволяет отображать текст UI на разных языках без изменения JSON-конфигурации. Все переводы хранятся в файле `Translations.xlsx`, который загружается при старте приложения. Смена языка применяется мгновенно ко всем элементам экрана.

**Поменять язык в картомате можно с помощью команды**
```"language:ключЯзыка"```

---

## Файл переводов Translations.xlsx

Файл представляет собой таблицу, где:

- **Первая строка** — заголовки: первая ячейка пустая (или служебная), далее идут ключи языков (`uz`, `ru`, `en` и т.д.)
- **Первый столбец** — ключи переводов (например, `Default.NextButton`, `otp.Title`)
- **Остальные ячейки** — тексты на соответствующем языке

Пример структуры:

| *(ключ)*           | `ru`             | `uz`            | `en`         |
|--------------------|-----------------|-----------------|--------------|
| `Default.NextButton` | Далее           | Keyingisi       | Next         |
| `otp.Title`        | Введите код     | Kodni kiriting  | Enter code   |
| `cardPrint.Title`  | Печатаем карту, {0}! | Kartani chop etamiz, {0}! | Printing card, {0}! |

### Пути к файлу

Система ищет файл в следующем порядке:
1. `%ProgramData%\KioskCommon\Translations.xlsx` (путь по умолчанию)
2. `<ResourcesFolder>\Translations.xlsx` (папка ресурсов из конфига приложения)

Если файл не найден — приложение создаёт флаг ошибки и продолжает работу с пустыми переводами (ключи отображаются как есть).

## LocalizationController

Синглтон `LocalizationController` отвечает за хранение всех словарей и выдачу переводов.

**Ключевые свойства:**

| Свойство | Описание |
|---|---|
| `SelectedLang` | Текущий активный язык (ключ языка, например `"ru"`) |
| `SelectedDictionary` | Словарь переводов для текущего языка |
| `LangDictionaries` | Все языки: `Dictionary<string, ObservableKeyDictionary>` |

**Ключевые методы:**

| Метод | Описание |
|---|---|
| `SetLanguage(string key)` | Переключает язык. Если ключ не найден — ничего не происходит |
| `SetDefaultLang()` | Устанавливает первый язык из файла |
| `Translate(string key, params object[] values)` | Возвращает перевод с подстановкой аргументов через `string.Format` |

**Поведение при отсутствии ключа:** если ключ не найден в словаре, `Translate` возвращает сам ключ как fallback.

## Язык по умолчанию

При инициализации приложения язык устанавливается из поля `defaultLanguage` корневого JSON-файла:

```json
{
  "defaultLanguage": "ru"
}
```

Он записывается в `PersistentData` под ключом `-language-`. Это позволяет элементам реактивно следить за сменой языка через `PersistentData.Watch("-language-", ...)`.

---

## Как элемент отображает перевод

При рендеринге label/button с `translationKey`:

1. Читается значение `translationKey` из JSON-определения элемента.
2. Если ключ начинается с `session:` — он резолвится динамически из `SessionData` (динамический ключ).
3. Из `customProperties` читаются `translationArguments` — список аргументов через запятую.
4. Вызывается `LocalizationController.Instance.Translate(key, args)`.
5. Результат парсится через `ParseHtmlLikeToInlines` и отображается в `TextBlock`.
6. Устанавливается реактивная подписка на `PersistentData["-language-"]` — при смене языка текст обновляется автоматически.

## Разметка текста в переводах

В строках переводов поддерживаются HTML-подобные теги форматирования:

| Тег | Описание |
|---|---|
| `<b>текст</b>` | Жирный текст |
| `<i>текст</i>` | Курсив |
| `<u>текст</u>` | Подчёркивание |
| `<color=#FF0000>текст</color>` | Цвет текста (HEX или название) |
| `\n` | Перенос строки |

Теги можно вкладывать: `<b><color=#FF0000>красный жирный</color></b>`

---

## Хранение выбранного языка

При смене языка командой `language:` текущий язык сохраняется в `PersistentData` под ключом `-language-`. Это означает, что язык сохраняется на всё время работы приложения (до перезагрузки) и не сбрасывается при переходах между экранами.
